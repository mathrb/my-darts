# X01 – Complete State Transition Tables

**Derived from:** `X01 (Standard Darts)`
**Status:** Authoritative (engine + server validation)

---

## 1. State Model (Explicit)

The following state fields are assumed to exist.
These are *derived state*, never directly mutated outside transitions.

### Per Game

* `starting_score`
* `in_strategy` ∈ {StraightIn, DoubleIn, MasterIn}
* `out_strategy` ∈ {StraightOut, DoubleOut, MasterOut}
* `legs_to_win`
* `current_leg_index`
* `game_complete`

### Per Leg / Player

* `score`
* `is_in` (bool)
* `legs_won`

### Per Turn

* `turn_active` (bool)
* `turn_start_score`
* `darts_thrown_in_turn` ∈ {0,1,2,3}
* `current_player`

---

## 2. Event Set (Relevant to X01)

* `GameCreated`
* `TurnStarted`
* `DartThrown`
* `TurnEnded`
* `LegCompleted`
* `GameCompleted`

---

## 3. Transition Tables

Each table is **orthogonal** and must be applied in order.

---

## Table A — Turn Start

| Current State  | Event       | Guard | Result                     |
| -------------- | ----------- | ----- | -------------------------- |
| No active turn | TurnStarted | —     | `turn_active = true`       |
| No active turn | TurnStarted | —     | `darts_thrown_in_turn = 0` |
| No active turn | TurnStarted | —     | `turn_start_score = score` |

**Invalid**

* TurnStarted while `turn_active == true` → reject

---

## Table B — DartThrown (General Acceptance)

| State Predicate  | Event      | Guard | Result |
| ---------------- | ---------- | ----- | ------ |
| Game complete    | DartThrown | —     | Reject |
| Turn inactive    | DartThrown | —     | Reject |
| DartsThrown == 3 | DartThrown | —     | Reject |

---

## Table C — In Strategy Resolution

### Applies **only if `is_in == false`**

| In Strategy | DartThrown              | Guard | Result                             |
| ----------- | ----------------------- | ----- | ---------------------------------- |
| Straight In | Any hit                 | —     | `is_in = true`, proceed to scoring |
| Double In   | Hit is double           | —     | `is_in = true`, proceed to scoring |
| Double In   | Hit not double          | —     | No score change                    |
| Master In   | Hit is double or triple | —     | `is_in = true`, proceed to scoring |
| Master In   | Hit single              | —     | No score change                    |

**Notes**

* Dart counts as thrown regardless of success
* Failure does NOT end the turn

---

## Table D — Scoring (Only if `is_in == true`)

Let:
`throw_value = segment × multiplier`
`resulting_score = score - throw_value`

| State | DartThrown | Guard                | Result                    |
| ----- | ---------- | -------------------- | ------------------------- |
| In    | DartThrown | resulting_score > 1  | `score = resulting_score` |
| In    | DartThrown | resulting_score < 0  | Bust                      |
| In    | DartThrown | resulting_score == 1 | Bust                      |
| In    | DartThrown | resulting_score == 0 | Proceed to Out Validation |

---

## Table E — Out Strategy Validation

Triggered **only when `resulting_score == 0`**

| Out Strategy | DartThrown              | Guard | Result       |
| ------------ | ----------------------- | ----- | ------------ |
| Straight Out | Any hit                 | —     | LegCompleted |
| Double Out   | Hit is double           | —     | LegCompleted |
| Double Out   | Hit not double          | —     | Bust         |
| Master Out   | Hit is double or triple | —     | LegCompleted |
| Master Out   | Hit single              | —     | Bust         |

---

## Table F — Bust Resolution

A **Bust** is triggered if:

* resulting_score < 0
* resulting_score == 1
* Out strategy validation fails

| State         | Event | Guard | Result                     |
| ------------- | ----- | ----- | -------------------------- |
| Bust occurred | —     | —     | `score = turn_start_score` |
| Bust occurred | —     | —     | Force TurnEnded            |

---

## Table G — Dart Count Increment

| State       | DartThrown | Guard | Result                      |
| ----------- | ---------- | ----- | --------------------------- |
| Turn active | DartThrown | —     | `darts_thrown_in_turn += 1` |

---

## Table H — Turn End Conditions

| State       | Event      | Guard             | Result    |
| ----------- | ---------- | ----------------- | --------- |
| Turn active | DartThrown | darts_thrown == 3 | TurnEnded |
| Turn active | Bust       | —                 | TurnEnded |

---

## Table I — Turn End

| State       | Event     | Guard | Result                   |
| ----------- | --------- | ----- | ------------------------ |
| Turn active | TurnEnded | —     | `turn_active = false`    |
| Turn active | TurnEnded | —     | Advance `current_player` |

---

## Table J — Leg Completion

| State        | Event | Guard                   | Result          |
| ------------ | ----- | ----------------------- | --------------- |
| LegCompleted | —     | —                       | `legs_won += 1` |
| LegCompleted | —     | legs_won < legs_to_win  | Reset Leg       |
| LegCompleted | —     | legs_won == legs_to_win | GameCompleted   |

---

## Table K — Leg Reset

Triggered after LegCompleted when match not finished.

| Action         | Result                                |
| -------------- | ------------------------------------- |
| Reset scores   | `score = starting_score (+ handicap)` |
| Reset in-state | `is_in = false`                       |
| Reset turn     | `turn_active = false`                 |
| Advance leg    | `current_leg_index += 1`              |

---

## Table L — Game Completion

| State     | Event         | Guard | Result                 |
| --------- | ------------- | ----- | ---------------------- |
| Match won | GameCompleted | —     | `game_complete = true` |

**Invariant**

* No DartThrown accepted after this point

---

## 4. Derived Invariants (Must Always Hold)

* `0 ≤ darts_thrown_in_turn ≤ 3`
* `score ≥ 0`
* Only one active turn at a time
* Bust always ends the turn
* Leg completion can only occur via `resulting_score == 0`
* In-strategy applies only until satisfied

---

## 5. Notes on Ambiguities (Explicitly Resolved)

The following interpretations are **required** for determinism:

1. A failed **In** dart still consumes a dart
2. Bust always immediately ends the turn
3. Out validation occurs **before** dart count increment is considered terminal
4. Bull (50) counts as a **double** for in/out validation
5. A miss (`segment = 0`) skips Tables C, D, and E entirely. It applies only Table G (dart count increment) and feeds into Table H (turn end condition check). It does not change score, does not satisfy in-strategy, and cannot cause a bust.

These are consistent with standard darts practice and your document.
