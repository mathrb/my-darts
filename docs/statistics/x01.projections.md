# X01 — Formal Statistics Projections

**Derived from:** Game Events + X01 Transition Tables
**Status:** Authoritative
**Scope:** Player, Leg, Match, Historical Analytics

---

## 1. Purpose

This document defines **all statistical views (projections)** for X01 games.

Projections:

* Are **purely derived** from events
* Never mutate game state
* May be rebuilt at any time by replay
* Are safe for offline, multiplayer, and corrections

> **No statistic is stored as a fact. Only events are facts.**

---

## 2. Projection Model Overview

Statistics are computed by replaying the following events:

* `GameCreated`
* `TurnStarted`
* `DartThrown`
* `DartCorrected`
* `TurnEnded`
* `LegCompleted`
* `GameCompleted`

Each projection defines:

* Input events
* Accumulated fields
* Reset rules
* Correction handling

---

## 3. Projection Categories

X01 statistics fall into **four orthogonal categories**:

1. Dart-level
2. Turn-level
3. Leg-level
4. Match-level

Each category can be recomputed independently.

---

## 4. Dart-Level Projections

### 4.1 Total Darts Thrown

**Purpose:** Baseline efficiency metric

**Inputs**

* `DartThrown`
* `DartCorrected`

**Accumulator**

* `total_darts_thrown += 1`

**Notes**

* Corrected darts still count as one dart
* Bust darts count

---

### 4.2 Scoring Darts

**Definition**
Darts that reduce the score (i.e., after in-strategy satisfied and not bust-reverted).

**Accumulator**

* Increment only if dart contributes to score

**Derived**

* Used for average calculations

---

### 4.3 Score per Dart

```text
score_per_dart = total_scored_points / scoring_darts
```

---

## 5. Turn-Level Projections

### 5.1 Turn Count

**Incremented On**

* `TurnEnded`

---

### 5.2 Average per Turn (3-Dart Average)

**Accumulator**

* `turn_score = turn_start_score - turn_end_score`

```text
three_dart_avg = (total_scored_points / total_darts_thrown) × 3
```

**Notes**

* Bust turns score = 0
* Partial turns still count as turns

---

### 5.3 Bust Rate

```text
bust_rate = bust_turns / total_turns
```

**Bust Detected When**

* Turn ended due to bust condition

---

## 6. Leg-Level Projections

### 6.1 Legs Won

**Incremented On**

* `LegCompleted` where competitor is winner
* …unless the originating game is **solo** (only one competitor). Solo games
  are practice / sandbox runs and are excluded from career-level legs played
  and legs won totals (see issue #106). Per-game stats (`getPlayerStatsForGame`)
  still count the leg.

---

### 6.2 Darts per Leg

```text
darts_per_leg = total_darts_thrown / legs_won
```

---

### 6.3 First-Dart In Success Rate

**Applicable Only If**

* In-strategy ≠ Straight In

**Accumulator**

* Attempted on first dart of leg
* Success if in-strategy satisfied by first dart

```text
first_dart_in_pct = successes / attempts
```

---

### 6.4 Checkout Attempts

**Detected When**

* Score ≤ 170
* Player is “in”

**Accumulator**

* Attempted when a turn begins in checkout range
* Successful on `LegCompleted`

---

### 6.5 Checkout Percentage

```text
checkout_pct = successful_checkouts / checkout_attempts
```

---

## 7. Match-Level Projections

### 7.1 Match Average

```text
match_avg = (total_scored_points / total_darts_thrown) × 3
```

---

### 7.2 Match Duration

**Derived From**

* First `TurnStarted`
* `GameCompleted`

---

### 7.3 Highest Checkout

**Tracked From**

* Final scoring dart in each leg

---

### 7.4 Highest Turn Score

**Computed Per Turn**

* Max of all turn_score values

---

## 8. Strategy-Specific Projections

### 8.1 Double-In Success Rate

**Applicable When**

* In-strategy = Double In or Master In

```text
double_in_success_pct = successful_in / total_in_attempts
```

---

### 8.2 Double-Out Success Rate

**Applicable When**

* Out-strategy includes double

```text
double_out_success_pct = successful_double_outs / checkout_attempts
```

---

## 9. Vision & Correction Handling

### Correction Rule

If a `DartCorrected` event occurs:

* All projections **from that dart forward** must be recomputed
* No special casing allowed

This is why projections must be replayable.

---

## 10. Projection Reset Rules

| Scope       | Reset On      |
| ----------- | ------------- |
| Dart stats  | Never         |
| Turn stats  | TurnEnded     |
| Leg stats   | LegCompleted  |
| Match stats | GameCompleted |

---

## 11. Explicit Non-Statistics

The following are **not projections**:

* Current score
* Turn dart count
* Active player
* Game legality

These belong to **game state**, not analytics.

---

## 12. Why This Design Is Correct

* Works offline
* Survives corrections
* Multiplayer-safe
* No race conditions
* Deterministic
* No hidden coupling to UI
